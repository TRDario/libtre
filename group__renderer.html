Detailed Description

Renderers and related functionality.

libtre prefers to work with self-contained renderers, as opposed to directly manipulating OpenGL state, which can cause inefficiencies if repeated, redundant state changes are made.

When making a tre-compatible renderer, setLastRendererID and lastRendererID should be used to facilitate more efficient state changes.

An instance of tr::Window must be created before any functionality from this section can be used.
