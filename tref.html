tre Bitmap Font Format (tref)

libtre uses a custom bitmap font format called tref: files of this type can be loaded with BitmapTextFormat.

To create a tref file, the trefc compiler is provided alongside the libtre source code. It is used as follows:

trefc [path to font information file] [path to image file] [path to output file]

FONT INFORMATION

The information file is a UTF-8 encoded text file consisting of the following:

  • A line skip definition: line_skip: [integer]
  • A newline-separated list of glyph definitions: [codepoint]: x: [integer], y: [integer], width: [integer], height: [integer], xoffset: [integer], yoffset: [integer], advance: [integer]
    • Where [codepoint] is one of: NUL, 0x[codepoint hex], '[codepoint char]'
    • line_skip is limited to a signed 32-bit integer value, x, y, width, height are limited to 16-bit unsigned integer values, and xoffset, yoffset, advance are limited to 16-bit signed integer values.
  • Any amount of empty lines and whitespace in between.

IMAGE FILE

trefc supports BMP, PNG, and JPEG image files.

EXIT CODES

0 - Success

1 - Invalid argument count

2 - File opening failure

3 - Parsing failure

4 - Image failure

5 - Writing failure

-1 - Unhandled exception
